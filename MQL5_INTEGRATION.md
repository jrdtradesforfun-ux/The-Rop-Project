# MQL5 EA Integration Guide

## Overview

The Jaredis Smart backend communicates with MetaTrader 5 Expert Advisors through a socket-based protocol. The EA receives trading signals and executes orders on the MetaTrader platform.

## Architecture

```
Python Backend (Trading Decisions)
    ↓ JSON/Socket
MetaTrader 5 (Order Execution)
    ↓
Broker Connection (Trade Execution)
```

## Signal Protocol

### Buy Order
```json
{
    "type": "order",
    "order_type": "buy",
    "symbol": "EURUSD",
    "entry": 1.0960,
    "stop_loss": 1.0910,
    "take_profit": 1.1010,
    "volume": 1.0,
    "comment": "Jaredis Smart Signal",
    "confidence": 0.85,
    "magic_number": 12345
}
```

### Sell Order

```json
{
    "type": "close",

```

```json
{
    "type": "modify",
    "ticket": 123456,
    "stop_loss": 1.0900,
    "take_profit": 1.1020
}
```

## EA Setup Steps

1. **Compile EA**: Open JaredisSmartEA.mq5 in MetaEditor and compile
2. **Attach to Chart**: Drag EA onto a chart in MT5
3. **Configure Parameters**:
   - PythonHost: Backend IP address
   - PythonPort: Backend listening port
   - MagicNumber: Unique identifier for orders
4. **Enable AutoTrading**: Ensure "AutoTrading" is enabled in MT5
5. **Test**: Monitor Experts tab for connection messages

## Backend Signal Sending

```python
from jaredis_backend.mql5_bridge import MQL5Connector, SignalCommunicator

# Connect to EA
connector = MQL5Connector(host="localhost", port=5000)
connector.connect()

# Create signal communicator
signal_comm = SignalCommunicator(connector)

# Send buy signal
signal_comm.send_buy_signal(
    symbol="EURUSD",
    entry=1.0960,
    stop_loss=1.0910,
    take_profit=1.1010,
    volume=1.0,
    comment="ML signal confirmed"
)
```

## Order Tracking

The backend track positions via:
1. Position ID (generated by backend)
2. MT5 Ticket number (sent back by EA)
3. P&L calculations based on entry/exit prices

## Error Handling

Connection failures:
- Backend automatically attempts reconnection
- Unsent signals are logged for manual review
- EA can continue accepting orders from other sources

Signal validation:
- SL must be on opposite side of entry from TP
- Volume must be positive
- Symbol must exist on connected broker

## Performance Considerations

- Socket communication is low-latency
- EA processes signals sequentially
- Multiple signals to same symbol override previous
- Use magic_number to track signal origin

## Production Deployment

For production:
1. Use secure VPN/proxy for network communication
2. Implement signal confirmation/acknowledgment
3. Monitor connection health continuously
4. Log all signals and executions
5. Set up redundant communication paths
6. Implement rate limiting on signal frequency

## Troubleshooting

### EA won't connect
- Check firewall rules
- Verify IP and port configuration
- Ensure Python backend is running
- Check EA logs in MT5 experts tab

### Orders not executing
- Verify symbol name matches broker
- Check account has sufficient margin
- Ensure trading hours are active
- Check for broker restrictions/holidays

### Partial execution
- Broker may have position limits
- Check account drawdown limits
- Verify lot size is valid
- Check spread doesn't make SL/TP impossible

## Security Notes

- Use firewalls to restrict socket connections
- Consider using encrypted channels for production
- Store credentials securely
- Monitor for unauthorized signal sources
- Implement IP whitelisting
- Use unique magic numbers to identify legitimate signals
